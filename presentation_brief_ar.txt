ملف ملخص مشروع (جاهز للإرسال لعمل عرض تقديمي)
اسم المشروع: Mechanic Assist (المساعد التقني)
المبتكر: أحمد الزهراني
مدة العمل التقريبية: 3 شهور

========================================
1) فكرة المشروع (Project Idea)
========================================
- مشروع “المساعد التقني” هو تطبيق جوّال يساعد المستخدم في تشخيص المشاكل وتقديم إرشادات عملية عبر محادثة ذكية.
- المشروع لا يخدم الميكانيكا فقط، بل يمكن الاستفادة منه أيضًا في:
  1) ميكانيكا السيارة (أعطال، صيانة، قطع غيار)
  2) كهرباء السيارة (بطارية/دينمو/فيوزات/حساسات/أعطال تشغيل وإنارة)
  3) تبريد السيارة (مكيف السيارة: ضعف تبريد/تسريب/أصوات/صيانة)
  4) كهرباء المنزل (قواطع/أحمال/أفياش/إنارة/أمان)
  5) تبريد المنزل (مكيفات: ضعف تبريد/تنظيف فلاتر/أعطال شائعة)
  6) مجال الحاسب (كمبيوتر): بطء/حرارة/تعريفات/مشاكل شبكة/حلول صيانة مبسطة

========================================
2) المشكلة التي يحلها المشروع
========================================
- كثير من المستخدمين يواجهون أعطال تقنية (سيارة/منزل/كمبيوتر) ولا يعرفون “من أين يبدأون”.
- البحث التقليدي يأخذ وقت ويكون مشتت، والمصطلحات أحيانًا معقدة.
- المشروع يقدم “مساعد” يعطي شرح مبسط + خطوات مقترحة لتحديد السبب والتصرف الصحيح.

========================================
3) الحل المقترح (Solution)
========================================
- تطبيق بسيط بواجهة عربية:
  - شاشة محادثة: تسأل عن المشكلة وتستقبل ردود جاهزة ومبسطة.
  - شاشة توليد صور: توليد صور/رسومات توضيحية لقطع/مكونات (خصوصًا قطع السيارات) حسب نوع السيارة والموديل والسنة واسم القطعة.

========================================
4) التقنيات المستخدمة (Tech Stack) — من ملفات المشروع
========================================
أ) تطبيق الجوال (Frontend)
- Expo + React Native
- AsyncStorage لحفظ سجل المحادثة محليًا على الجهاز
- واجهة داكنة (Dark UI) ودعم عربي

ب) السيرفر (Backend)
- Node.js + Express
- CORS مفتوح لتسهيل الاتصال من أي جهاز/شبكة أثناء التجارب
- OpenAI SDK
  - مسار /chat للمحادثة (نموذج: gpt-3.5-turbo)
  - مسار /image لتوليد الصور (نموذج: dall-e-2)

ج) الاستضافة والنشر
- ربط التطبيق بسيرفر مستضاف على Render عبر رابط API ثابت.

د) Firebase (تجهيز مستقبلي)
- يوجد إعداد Firebase (Database + Storage) عبر متغيرات بيئة Expo
- الهدف منه: قابلية إضافة ميزات مستقبلية مثل حفظ بيانات/صور سحابيًا بدل المحلي فقط.

========================================
5) المزايا الأساسية (Core Features)
========================================
1) محادثة ذكية
- المستخدم يكتب المشكلة.
- التطبيق يرسل الرسالة للسيرفر (POST /chat).
- السيرفر يرسلها للذكاء الاصطناعي مع توجيه أنه “خبير” في المجال.
- يرجع الرد للمستخدم داخل المحادثة.

2) حفظ سجل المحادثة + الحذف (Deletion)
- حفظ سجل المحادثة محليًا (AsyncStorage).
- من قائمة النقاط الثلاث (⋮):
  - محادثة جديدة: تفريغ الرسائل من الشاشة لبدء جلسة جديدة.
  - مسح المحادثة: تفريغ الرسائل + حذف السجل من التخزين المحلي (حذف فعلي لبيانات الجهاز).

3) توليد الصور (Image Generation)
- شاشة إدخال: نوع السيارة + الموديل + السنة + اسم القطعة.
- إنشاء Prompt وإرساله للسيرفر (POST /image).
- السيرفر يقوم بتعديل/تحسين الطلب:
  - تصحيح بعض الأخطاء الإملائية الشائعة في اسم السيارة/القطعة.
  - توليد Prompt مفصل يطلب رسمة تقنية (Line Art) ويشدد على “عزل القطعة فقط” بدون إظهار السيارة كاملة.
- النتيجة ترجع كرابط صورة ويتم عرضها في التطبيق.

4) حفظ الصورة في المعرض (ملاحظة)
- يوجد زر “حفظ في المعرض” في الواجهة.
- حالياً يظهر رسالة نجاح، ويُعتبر جاهز كواجهة ويحتاج إكمال برمجي للحفظ الفعلي على الجهاز (ميزة مستقبلية/قيد التطوير).

========================================
6) خطوات العمل خلال 3 شهور (Work Plan Timeline)
========================================
المرحلة 1: التحليل والتخطيط (تقريباً 2–3 أسابيع)
- تحديد نطاق المشروع (مساعد تقني متعدد المجالات).
- تصميم تدفق المستخدم (محادثة + قائمة + توليد صور).
- اختيار التقنية (Expo) لتسريع تطوير تطبيق الجوال.

المرحلة 2: بناء السيرفر والربط مع الذكاء الاصطناعي (تقريباً 3–4 أسابيع)
- إنشاء API بـ Express لمسارات /chat و /image.
- إعداد متغيرات البيئة لمفاتيح OpenAI.
- إعداد CORS لتسهيل الاختبار على الجوال.
- تجربة الاستجابات وإدارة الأخطاء.

المرحلة 3: بناء تطبيق الجوال والواجهات (تقريباً 4–5 أسابيع)
- تصميم واجهة المحادثة (فقاعات + تحميل + إدخال).
- حفظ واسترجاع السجل محليًا (AsyncStorage).
- إضافة قائمة خيارات (⋮) + (محادثة جديدة/مسح محادثة).
- بناء شاشة توليد الصور وعرض النتائج.

المرحلة 4: تحسين التجربة + الاستقرار + النشر (تقريباً 2–3 أسابيع)
- إضافة Timeouts وإلغاء الطلب عند التأخر.
- رسائل واضحة للمستخدم عند تأخر السيرفر/ضغط.
- ربط التطبيق بسيرفر مستضاف على Render.

========================================
7) التعب والصعوبات (Challenges)
========================================
- Cold Start على Render:
  - السيرفر قد يتأخر في أول تشغيل (حوالي 30 ثانية) مما يسبب شعور المستخدم أن التطبيق “عطل”.
  - تم معالجة ذلك برسالة تنبيه وإرشاد للمحاولة بعد قليل.

- إدارة الوقت والمهل (Timeouts):
  - في المحادثة مهلة أطول بسبب انتظار الرد.
  - في توليد الصور مهلة أقصر لأن التوليد قد يتأخر.

- مشاكل الاستجابة غير المتوقعة:
  - في توليد الصور تم التحقق من نوع الاستجابة (JSON) لأن السيرفر أحياناً يرجع محتوى غير متوقع أثناء التشغيل.

- جودة نتائج الصور (Prompt Engineering):
  - التحدي كان الحصول على صورة “قطعة فقط” بدون سيارة كاملة.
  - تم تشديد التعليمات داخل الـ prompt لعزل القطعة وتحويلها لرسمة تقنية واضحة.

- دعم العربية/الأسماء:
  - إضافة تصحيح لبعض الكلمات الشائعة لتقليل أخطاء الإدخال وتحسين جودة النتائج.

========================================
8) الإضافات والتحسينات (Enhancements)
========================================
- قائمة (⋮) لإدارة المحادثة والانتقال لتوليد الصور.
- حفظ سجل محادثة محليًا واسترجاعه عند فتح التطبيق.
- شاشة مستقلة لتوليد صور قطع الغيار.
- سكربت مساعد لتحويل شعار المشروع إلى PNG (تجهيز الأيقونة والسلاش).

========================================
9) اقتراح ترتيب الشرائح (Slide Outline)
========================================
1) عنوان المشروع + اسم المبتكر + مدة العمل (3 شهور)
2) المشكلة التي يحلها المشروع
3) فكرة الحل + المجالات المستفيدة (سيارة/منزل/كمبيوتر)
4) نظرة عامة على التطبيق (محادثة + توليد صور)
5) التقنيات المستخدمة (Expo/React Native + Express + OpenAI + Render)
6) كيف تعمل المحادثة (Flow)
7) كيف يعمل توليد الصور (Flow + Prompt)
8) الحفظ والحذف (AsyncStorage + مسح محادثة)
9) التحديات (Cold start/Timeout/Prompt)
10) خطة العمل خلال 3 شهور (مراحل)
11) تحسينات مستقبلية (حفظ فعلي للصورة، استخدام Firebase، توسعة المجالات)
12) خاتمة + أسئلة
